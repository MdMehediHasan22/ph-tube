Discuss the scope of var, let, and const

1. Scope
var: Has function scope, meaning it is scoped to the function in which it is declared. If declared outside a function, it becomes globally scoped, accessible throughout the entire script.
let and const: Both have block scope, meaning they are only accessible within the nearest set of curly braces ({}) such as within loops, conditionals, or functions.
2. Reassignability
var: Can be re-assigned and re-declared within the same scope, which can sometimes lead to bugs if the same name is reused unintentionally.
let: Can be re-assigned but cannot be re-declared in the same scope.
const: Cannot be re-assigned or re-declared in the same scope. This is ideal for values that do not need to change, such as configuration settings or constant values.
3. Hoisting
var: Is hoisted to the top of its scope (function or global), meaning the variable can be accessed before it’s declared, though it will initially be undefined.
let and const: Are also hoisted, but in a "Temporal Dead Zone" from the start of the block until their declaration is encountered. Attempting to access these variables before their declaration results in a ReferenceError.


function testScope() {
// var
if (true) {
var a = 10;
}
console.log(a); // 10, because `var` is function-scoped
// let
if (true) {
let b = 20;
}
console.log(b); // ReferenceError, because `let` is block-scoped
// const
if (true) {
const c = 30;
}
console.log(c); // ReferenceError, because `const` is block-scoped
}

Understanding null vs undefined in JavaScript

When working in JavaScript,we'll often encounter null and undefined, both of which represent the absence of a value. However, they serve different purposes and are used in distinct contexts.
Let’s break down when and why to use each one.


null

When working in JavaScript, you'll often encounter null and undefined, both of which represent the absence of a value. However, they serve different purposes and are used in distinct contexts. Let’s break down when and why to use each one.
Intentional Empty Value: Use null when you want to signify that a variable is intentionally empty and may later hold an object or value.
Object Property Reset:: Set a property to null to reset or empty its value, often in situations where undefined might have a different semantic meaning.
API or Database Responses: :When expecting data from APIs or databases, null is commonly used to signify "no data" or "empty value".
DOM Element Placeholders: In front-end development, null is used to initialize variables intended to store references to DOM elements once they are loaded.

undefined

Definition: undefined means a variable has been declared but not assigned a value. It is also the default value of function parameters that are not provided and the return value of functions that do not explicitly return anything.
Uninitialized Variables: When you declare a variable without assigning a value, JavaScript automatically assigns it undefined.
Function Parameters Not Provided:: If a function is called without passing all arguments, the missing parameters are assigned undefined.
Non-Existent Properties :Accessing a non-existent property on an object returns undefined.
Function Return Value: If a function does not have a return statement, it implicitly returns undefined.

What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services. It allows different software applications to communicate with each other over the internet. REST APIs are commonly used in web development to enable interactions between front-end applications (like web or mobile apps) and back-end services (such as databases or servers).

Key Concepts of REST API

REST Architecture: : REST is an architectural style based on a few principles that make web services simpler and more efficient:
1. Stateless: Each request from a client to a server must contain all the information the server needs to fulfill that request. The server does not store any client context between requests, making each interaction independent.
2. Client-Server :The client (e.g., web browser or mobile app) interacts with the server (e.g., a database or application server) to request resources, without needing to know how the server stores or manages data.
3. Uniform Interface:REST APIs use a standardized way of accessing resources, typically using HTTP methods like GET, POST, PUT, DELETE, etc.
4. Cacheable:Responses from REST APIs can be cached on the client side, which improves performance and reduces server load.


HTTP Methods: : REST APIs use HTTP methods to perform various operations on resources:
1. GET: Retrieve information or data from the server.
2. POST: Create a new resource on the server.
3. PUT:Update an existing resource on the server.
4. DELETE:Remove a resource from the server.


Resources and Endpoints: In REST, "resources" are the objects or data that you want to interact with, like users, products, or orders. Each resource is accessible through a unique URL called an "endpoint," such as /users or /products/123.


JSON and XML: REST APIs often use JSON (JavaScript Object Notation) to format the data sent between the client and server, though XML is also sometimes used. JSON is human-readable and easy for front-end applications to parse, making it a preferred format.
